## Branch and commit prefix rules

Rules bellow are recommended for creating branches and commits' name.  
There can be hiearachy with these rules for branches and commits.  
For example, `fix/bug-no-1` branch can contains `fix: useEffect's wrong dependency` commit.

- post: Posting new content.
- fix: Fixing bugs or possible errors.
- layout: Creating new layout like header, nav, etc...
- feat: Developing new feature.
- add: Adding something to existing one.
- chore: Edit something not very important but need to be fixed.
- update: Update version of dependencies or adding more content to existing one.
- refactor: Refactor the code to a better version and with same result.
- revert: Reverting existing commits or merged PRs.
- docs: Updating docs like `README.md`
- create: Create new component or others.
